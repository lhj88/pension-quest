## Language
- 모든 응답은 한국어로 작성한다.
- 코드, 명령어, 파일 경로, 에러 메시지, 로그는 원문 그대로 유지한다.

## Working Style
- 요구사항이 모호하면 가정 1~2개를 명시하고 진행한다.
- 변경은 최소 범위로 적용하고, 근본 원인을 우선 해결한다.
- 수정 후에는 가능한 가장 작은 단위 테스트/검증부터 실행한다.
- 실패나 제약이 있으면 즉시 보고하고 우회안을 함께 제시한다.

## Output Format
- 결론을 먼저 제시하고, 필요한 근거를 짧게 덧붙인다.
- 파일 변경 설명에는 파일 경로와 핵심 변경점을 함께 제시한다.
